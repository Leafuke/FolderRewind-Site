## FolderRewind v1.7.0

v1.7.0 is a major update. It brings a complete cloud-archive workflow, stronger automation controls, and a series of reliability and UX improvements.

<!-- truncate -->

:::tip Download advice
It is recommended to install FolderRewind from [Microsoft Store](https://apps.microsoft.com/detail/9nwsdgxdqws4) first.
:::

## Update Content

1. Added full cloud-archive workflow support. See the [Cloud Archive Guide](/en/docs/guides/cloud-archive).
2. Auto backup now supports selected targets instead of always backing up the entire config.
3. Added condition-based backup mode (for example, auto backup after game exit).
4. Added more template path-rule support.
5. Added settings-page search.
6. Added runtime status display.
7. Improved visual presentation on selected pages.
8. Fixed core feature auto-validation failures.
9. Fixed `LIST_WORLDS` remote command parsing issues.
10. Other minor bug fixes.

## Highlights

### Cloud archive workflow is now complete

v1.7.0 makes “local backup + cloud redundancy” much easier to deploy:

- use rclone to sync FolderRewind backup directories to OneDrive and similar providers
- follow a complete initial setup guide (`rclone config`)
- improve resilience for multi-device and off-device scenarios

Cloud features are now directly available from:

- post-backup auto upload in Config Settings
- "Sync this config from cloud" in Config Settings
- per-item upload/download actions in History page

Start here:

- [Cloud Archive Guide](/en/docs/guides/cloud-archive)

### Automation becomes more precise

Automation is upgraded in v1.7.0:

- choose selected auto-backup targets
- use condition-based triggers (for example, after game exit)

This makes it easier to combine high-frequency protection for critical data with lower-frequency tasks for secondary data.

### Template and settings experience improvements

This release also improves daily maintainability:

- broader template path-rule coverage
- settings search support
- runtime status visibility in settings
- page-level UI refinements

### Reliability fixes

v1.7.0 also resolves several important issues:

- core feature auto-validation failure
- `LIST_WORLDS` parsing failure in remote command workflow
- multiple minor known issues

## Recommended after upgrading

- validate selected-target and condition-based automation in a test config first
- complete one `rclone config` setup and run a minimal sync verification
- if you use Minecraft integration, retest `LIST_WORLDS` once after upgrade
