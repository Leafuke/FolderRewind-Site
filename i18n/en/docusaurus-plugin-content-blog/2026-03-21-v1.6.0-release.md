## FolderRewind v1.6.0

v1.6.0 focuses on a backup experience that is easier to validate, reuse, and share. It introduces automatic core feature validation, a full config template workflow, and stronger template sharing support.

<!-- truncate -->

:::tip Download advice
It is recommended to install FolderRewind from [Microsoft Store](https://apps.microsoft.com/detail/9nwsdgxdqws4) first.

If you use the side-loaded build, v1.6.0 now supports switching GitHub mirror sources to improve updates and online template access.
:::

## Update Content

1. Added support for automatic core functionality verification.
2. Added support for creating and using configuration templates.
3. Added support for sharing and importing templates.
4. Added support for setting GitHub mirror source.
5. Added support for automatic sidebar collapsing.
6. Optimized the display effects of certain interfaces.
7. Improved loading performance on some pages.
8. Enhanced the update experience for sideloaded versions.
9. Fixed a potential path out-of-bounds issue.
10. Fixed animation issues with setting cards.
11. Fixed the issue where safe deletion failed.
12. Other minor bug fixes.

## Highlights

### Automatic core feature validation

Settings now includes automatic core feature validation. It creates a temporary workspace and validates:

- initial full backup
- no-change skip
- smart incremental backup
- shared-lock backup
- clean restore and overwrite restore
- safe delete
- keep-count pruning

This is especially useful after upgrading, moving to a new machine, or before relying on automation.

### Config template workflow

You can now save proven configs as templates and then:

- create configs from templates
- auto-discover candidate source folders
- manage local templates
- reuse the same backup strategy across devices

This makes FolderRewind much more practical for multi-project, multi-device, and community-driven scenarios.

### Template sharing and mirror sources

v1.6.0 completes the path for exporting, importing, browsing official templates, importing by share code, and submitting templates through GitHub.

At the same time, side-loaded builds can now configure GitHub mirror sources, which also helps online template access.

## Recommended after upgrading

- run **Automatic Core Feature Validation** once in Settings
- create a template baseline for your most important config
- if you use the side-loaded build, review whether a GitHub mirror source would help
