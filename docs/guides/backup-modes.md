# 备份模式详解

FolderRewind 基于 7-Zip 引擎提供三种核心备份模式：**全量**、**智能增量**、**覆写**。

:::caution v1.5.0 兼容性提示
当前版本改进了增量备份与还原的工作模式。若你是从更早版本升级而来，请先用测试数据重新验证备份链和还原结果，再继续生产使用。
:::

## 先看结论

- 新手默认：**全量备份** + 开启“无变更时跳过备份”。
- 日常高频：**智能增量** + **链长上限** + **安全删除**。
- 临时快速更新：**覆写备份**（只维护最近一个包）。

## 备份模式对比

| 模式 | 说明 | 优势 | 适用场景 |
|------|------|------|---------|
| **全量备份 (Full)** | 每次生成一个完整新包 | 结构直观，还原稳定 | 新手、关键里程碑 |
| **智能增量 (Incremental/Smart)** | 仅打包变更文件，形成改进的增量链 | 速度快、占用低 | 大目录高频备份 |
| **覆写备份 (Overwrite)** | 更新最近备份包，不持续新增历史包 | 节省空间，操作简单 | 临时同步、缓存型目录 |

:::note 说明
当前产品文案中的“智能增量”对应代码中的 Incremental/Smart 语义。

在 v1.5.0 中，增量链的生成、截断与还原路径都有所调整，因此升级旧配置后建议重新做一轮验证。
:::

## 如何设置备份模式

1. 进入目标配置的 **配置设置**。
2. 打开 **备份策略**。
3. 在 **备份模式** 下拉框中选择：
	- 全量备份（每次生成新文件）
	- 智能增量（仅备份变更文件）
	- 覆写备份（更新最近的备份）
4. 点击保存并执行一次手动备份验证。

![备份模式下拉框三种选项](/img/docs/guides/backup-mode-dropdown-options.png)

## 选择建议

- **Minecraft 存档：** 智能增量（高频、体量大时更省空间）
- **工作文档/项目：** 智能增量或全量（看磁盘与恢复偏好）
- **重要资料归档：** 全量 + 加密配置
- **临时目录/缓存目录：** 覆写备份

## 与模式强相关的关键参数

在“配置设置 → 备份策略”中建议同时关注：

- **无变更时跳过备份**：避免产生无意义备份。
- **保留最近备份数**：`0` 表示不限制。
- **智能备份链长度上限**：仅对智能增量有意义；达到阈值后会触发一次全量截断链条。
- **安全删除**：清理增量链时尽量避免链断裂导致不可还原。
- **远程强制完整备份**：在联动场景中，可通过远程命令临时绕过当前模式，直接执行一次 Full 备份。

## 压缩与性能建议

FolderRewind 使用 7z/zip，可调压缩等级、算法和线程数：

- **压缩等级**：越高体积越小，但更慢。
- **CPU 线程数**：`0` 表示自动；性能不足时可限制线程。
- **文件类型规则**：可对 `*.zip`、`*.mp4` 等已压缩文件设置较低压缩等级，减少耗时。

## 首次选型建议（可直接照抄）

- 通用办公：全量 + 无变更跳过 + 保留 30
- 开发项目：智能增量 + 链长上限 20 + 安全删除 + 保留 60
- 大型游戏存档：智能增量 + 无变更跳过 + 自动化任务 + 定期验证还原
