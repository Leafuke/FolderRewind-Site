# 模板：分享与导入

v1.6.0 为模板补齐了完整的分享链路。你可以在本地导出模板，也可以导入别人分享的模板，或者直接浏览官方模板列表。

## 本地导出模板

在设置页的 **模板数据** 区域，可以选择：

- **导出模板**
- **导入模板**

导出时，你可以从本地模板列表中选择一项，保存为模板文件，方便：

- 跨设备迁移
- 团队内部共享
- 自己做版本留档

## 导入模板

导入模板时，程序会先检查模板文件是否合法，并处理同名冲突。

当发现冲突时，一般会给出两种选择：

- 覆盖现有模板
- 保留两份，并自动重命名

如果你不确定哪份更新，建议先保留两份，再进入模板管理器比较内容。

## 官方模板与分享码

除了手动导入文件，FolderRewind 还支持：

- 浏览官方模板列表
- 通过 **分享码** 导入模板

这适合游戏社区、插件作者或教程作者把模板分发给其他人。

## 准备模板分享包

如果你想把模板提交到官方模板仓库，或先发给别人预览，可以在设置页中准备模板分享包。

这一步通常会先做校验，确保模板适合公开分享，例如：

- 路径规则没有明显的危险路径
- 模板数据没有包含不适合共享的本地敏感信息
- 必要字段完整

## 直接提交到 GitHub

v1.6.0 还支持从应用内直接把模板提交流程发往 GitHub。

大致流程是：

1. 选择一个本地模板
2. 补充游戏名等分享信息
3. 登录 GitHub
4. 由应用生成并提交分享内容
5. 获取分享码和 Pull Request 链接

这使得“做出模板”到“把模板共享出去”之间的距离更短。

## 镜像源与在线模板

在线模板列表与模板文件下载会受到 **GitHub 源 / 镜像源** 设置影响。

如果你发现：

- 官方模板搜索很慢
- 模板文件下载失败
- 在线模板列表刷新异常

可以先到设置页切换镜像源后再试。

## 分享模板前的建议

- 先在另一台机器或另一份测试目录上试导入一次
- 尽量写清楚模板适用的场景和目录结构
- 如果依赖插件，明确写出插件名称
- 不要把个人私有目录结构硬编码进模板
