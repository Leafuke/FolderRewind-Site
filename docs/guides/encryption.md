# 加密备份

FolderRewind 通过 **Encrypted 配置类型**提供加密备份能力。创建后，备份与还原会按加密流程处理。

## 适用场景

- 备份目录包含隐私文档或账号数据
- 设备多人共用，需降低误访问风险
- 需要在本机长期保存加密归档

## 创建加密配置

1. 在首页点击 **新建配置**。
2. 配置类型选择 **Encrypted**。
3. 输入配置名，设置密码并确认。
4. 进入管理页，添加文件夹并执行备份。

:::caution 密码策略
当前版本中，加密密码在创建时设置后不可直接修改。
请务必妥善保存密码。
:::

## 还原时的行为

- 还原加密配置时，会先弹出密码验证。
- 验证通过后才会执行解压还原。
- 验证失败将中止还原。

## 存储与安全说明

- 加密密码不会明文写入配置 JSON。
- 密码使用 Windows DPAPI（当前用户范围）加密后本地存储。
- 同一台机器、同一用户账户环境下可正常校验与使用。

## 与其它策略配合建议

- 与 **还原前自动备份** 一起使用，降低误操作风险。
- 与 **自动化任务** 一起使用，形成持续加密快照。
- 大目录建议改用智能增量，平衡安全与性能。

## 常见问题

### 忘记密码怎么办？

当前无“找回明文密码”机制。建议：

- 通过密码管理器保存
- 在切换策略前先做一轮明文可验证备份

### 导入配置后为什么还原失败？

仅导入 `config.json` 不等于导入本机加密密码存储。
如跨设备迁移加密配置，请先在新设备做还原验证。
