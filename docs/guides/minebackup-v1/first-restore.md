# 首次还原

建议先在测试世界演练，不要直接对主存档做第一次还原。

一个稳妥原则：第一次还原只做“可验证的微小回滚”，先验证流程，再处理真实事故。

## 标准还原流程

1. 打开历史记录，选中目标备份
2. 选择还原方式并确认
3. 等待还原结束，重新进入世界验证

## 还原前准备清单

执行还原前，建议先确认：

1. 当前目标世界未处于高风险写入状态
2. 目标备份文件真实存在且可读取
3. 已知晓本次还原范围（整包/部分）
4. 已决定是否启用“还原前备份”兜底

## 还原方式说明

- Clean：清理后恢复，适合完整回退
- Overwrite：覆盖式恢复
- Reverse：反向策略（用于特定差异场景）
- Custom：自定义还原条目列表

### 如何选还原方式

- 不确定时：优先 `Clean`
- 想尽量保留现有目录结构：尝试 `Overwrite`
- 只恢复指定文件：使用 `Custom`

如果是首次演练，不建议直接使用 `Reverse`。

## 还原前备份

如果开启“还原前备份”，系统会先做一份兜底备份，再执行还原。

这项设置非常建议常开，尤其是生产世界。

## 成功标准

- 目标时间点状态恢复正确
- 关键文件可读取
- 历史记录中仍可追溯此次操作

## 还原后验证建议（3 分钟）

- 进入世界确认关键位置方块状态
- 检查最近的关键进度点是否符合预期
- 若启用了玩家数据相关联动，确认玩家状态是否正常

## 失败后的回退策略

如果本次还原结果不符合预期：

1. 立即停止继续操作
2. 使用“还原前备份”回到还原前状态
3. 改用更明确的目标备份重新执行
