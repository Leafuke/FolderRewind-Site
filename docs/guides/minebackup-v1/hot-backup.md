# 热备份与快照机制

当世界文件被占用（例如 `level.dat` 锁定）或你主动启用热备份时，MineBackup 会优先走快照/联动链路。

热备份的核心价值是：尽量在“游戏运行中”仍拿到可用备份。

## 热备份核心流程

1. 检测文件占用或热备份开关
2. 需要时与联动模块握手并等待保存完成
3. 创建临时快照目录
4. 对快照执行压缩备份
5. 结束后清理快照

## 什么时候会触发热备份

- 检测到关键文件锁定
- 配置中主动启用热备份
- 联动链路要求先保存再备份

## 关键路径参数

- `snapshotPath`：快照目录（可指定）
- 未指定时会回退系统临时目录

## 与联动模块配合时的建议

- 优先确认握手成功后再执行关键备份
- 出现超时时，不要连续重试高频命令
- 先回到手动备份链路确认基础功能可用

## 使用建议

- 游戏运行中做备份时建议开启
- 盘符空间不足时先检查快照路径
- 若联动超时，优先切回手动演练链路

## 常见失败点

- 快照目录无写权限
- 快照盘空间不足
- 联动端未按时回传保存完成信号

处理原则：先保证单次稳定成功，再恢复自动化或热链路。
