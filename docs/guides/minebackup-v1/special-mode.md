# 特殊模式（Special Config）

特殊配置用于自动化批处理场景，适合“打开即执行任务、执行完自动退出”。

如果你在做无人值守备份，这是一代里最关键的能力之一。

## 典型能力

- `autoExecute`：启动后自动执行
- 命令列表（旧版兼容）
- 自动备份任务（旧版兼容）
- 统一任务系统（新版）
- `exitAfterExecution`：完成后自动退出

## 典型应用场景

- 每天固定时间自动备份后自动退出
- 启动后先执行命令再执行备份
- 多个世界按顺序批量备份

## 适用场景

- 夜间批量备份
- 开机后无界面执行
- 多配置串行任务

## 配置建议

- 先在 GUI 手动跑通一次
- 再切换自动执行
- 对关键任务保留日志并开启异常通知

## 推荐落地步骤

1. 新建 Special Config
2. 加入 1 个最小 Backup 任务
3. 手动执行验证
4. 打开 `autoExecute`
5. 视需要打开 `exitAfterExecution`

## 常见误区

- 一开始就堆很多并行任务
- 没做手动验证直接自动运行
- 没有日志落盘，失败后无法排查

建议先做“最小可用自动化”，再逐步加复杂度。
