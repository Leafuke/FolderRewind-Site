# 首次备份

首次备份的核心目标是“验证你的配置真的可执行”，而不是追求最高压缩率。

## 基本操作

1. 在主界面选中目标世界
2. 输入可选注释（建议标记测试时间）
3. 触发备份并观察控制台日志

建议注释写法：`baseline_YYYYMMDD_HHMM`，后续回溯会更清晰。

## 备份过程中会发生什么

- 校验目标路径与压缩程序可用性
- 按当前模式决定 Full/Smart/Overwrite 流程
- 在需要时创建快照（热备份路径）
- 写入历史记录条目

## 你应该关注的日志节点

只看这几类信息即可快速判断是否成功：

- 备份开始头部日志
- 备份目录与压缩程序确认日志
- （可选）热备份快照创建日志
- 备份结束头部日志

如果中途出现外部命令失败或文件占用提示，优先检查热备份设置与路径权限。

## 你应重点确认的结果

- 备份文件真实出现在 `backupPath/世界名/`
- 历史记录窗口能看到新条目
- 若开启“无变更跳过”，二次备份会按预期跳过

## 备份后建议立刻做的两件事

1. 在历史记录里给这次备份加上可识别注释
2. 打开备份目录确认压缩文件体积正常（不是 0 KB 或异常小）

这两步能避免“看似成功但实际不可用”的假成功。

## 常见首次失败原因

- `7z.exe` 路径无效
- 备份目录不可写
- 世界被占用且热备份链路未准备好

## 首次失败时的最短恢复策略

1. 把模式先切回 Full
2. 暂时关闭复杂过滤规则
3. 再跑一次手动备份

先拿到一次稳定成功，再逐步打开 Smart、过滤、自动化。
