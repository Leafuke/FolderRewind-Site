# 创建第一套配置

MineBackup 支持多配置并行。每个配置可理解为一套独立规则集（路径、压缩、模式、任务）。

如果你同时管理多个启动器或多个存档根目录，建议从第一套配置开始就保持“一个配置只对应一类世界”的习惯。

## 配置中最关键的字段

- `saveRoot`：世界根目录
- `worlds`：世界列表（名称 + 描述）
- `backupPath`：备份输出目录
- `zipPath` / `zipFormat` / `zipMethod` / `zipLevel`
- `backupMode`、`keepCount`、`skipIfUnchanged`

## 创建前先决定一件事：按什么维度拆配置

推荐拆分方式：

- 按启动器拆：官方启动器 / 第三方启动器
- 按用途拆：正式生存档 / 测试整合包
- 按实例拆：单机 / 小型服

不要让多个配置把同名世界写入同一个备份目录，这会大幅提高误覆盖风险。

## 推荐创建流程

1. 新建普通配置（Normal Config）
2. 设置 `saveRoot` 与 `backupPath`
3. 扫描世界目录，确认世界列表
4. 先保留默认压缩参数，执行一次验证备份

## 建议的首套配置参数（稳妥版）

- `backupMode`：Full（先求稳）
- `skipIfUnchanged`：开启
- `keepCount`：30（可按磁盘空间调整）
- `zipLevel`：中等等级（默认即可）

这套参数的目标是：先建立稳定闭环，再优化体积与性能。

## 世界扫描注意点

- 扫描会读取 `saveRoot` 下的目录项
- 世界描述可用于 UI 区分，不建议留空太多
- 多启动器环境建议拆成多配置，避免目录冲突

补充建议：

- 世界描述建议写“版本 + 模组包 + 目的”，后续检索更直观
- 不打算备份的临时世界可在描述里标注，减少误操作

## 完成标准

- 至少存在 1 个世界
- 备份目录可写入
- 压缩程序路径有效

## 额外验证（可选但推荐）

完成配置后，立即做一次“改一个文件再备份”的最小验证：

1. 进入目标世界并做一个可见改动
2. 退出游戏（或保持运行并准备热备份）
3. 执行备份并确认历史新增条目

若这一步成功，你的配置基本可进入日常使用阶段。
