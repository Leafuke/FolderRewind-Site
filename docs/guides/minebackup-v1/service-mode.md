# 服务模式（Windows）

MineBackup 包含 Windows 服务模式配置与安装接口，适用于长期无人值守任务。

服务模式建议在“普通模式稳定运行一段时间”后再启用。

## 关键能力

- 安装/卸载服务
- 启动/停止服务
- 查询服务安装状态与运行状态
- 支持自动启动与延迟启动配置

## 推荐启用流程

1. 先在普通模式验证任务成功率
2. 固化任务和日志路径
3. 安装服务并设置启动策略
4. 观察 2~3 天服务运行稳定性

## 适用场景

- 固定机器上的定时备份
- 需要后台常驻执行任务

## 运维建议

- 为服务运行账号预留目录读写权限
- 固定日志输出位置，便于巡检
- 保留人工应急入口（可手动停服与切回 GUI）

## 边界说明

- 服务模式主要面向 Windows
- 非 Windows 平台相关接口为占位实现

## 风险提醒

- 服务模式下前台提示较少，错误更依赖日志
- 任务配置错误可能长期重复失败而不易察觉

建议至少配置“每日一次人工检查”。

## 建议

- 先在普通模式验证任务稳定后再服务化
- 明确日志位置，便于故障定位
