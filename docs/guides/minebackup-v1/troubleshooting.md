# 故障排查

建议按“先基础、再高级”的顺序排查：

1. 路径与权限
2. 压缩程序可用性
3. 备份模式与链完整性
4. 联动通信
5. 自动化任务

## 1) 找不到压缩程序

现象：备份开始后立即报错，提示无法找到 `7z.exe`。

处理：在设置中重新指定压缩程序路径，并执行一次手动备份验证。

补充：如果你使用的是便携目录，移动程序后请重新确认压缩程序路径。

## 2) 游戏运行中备份失败

现象：提示文件占用或快照失败。

处理：

- 开启热备份
- 配置可写的 `snapshotPath`
- 若使用联动，确认 KnotLink 链路可用

优先策略：先验证“退出游戏后的普通备份”是否成功，再回到热链路。

## 3) Smart 备份无法还原

现象：提示基准 Full 丢失或链异常。

处理：

- 检查历史中最近 Full 是否仍存在
- 重新做一次 Full 截断链条

如果历史链长期混乱，建议新建配置并重新建立备份基线。

## 4) 自动任务未按预期执行

现象：间隔任务不触发或触发后无输出。

处理：

- 检查任务是否启用
- 校验配置索引与世界索引有效性
- 先用单任务模式复现

补充检查项：

- 触发模式是否符合当前时间窗口
- 并行任务是否造成资源竞争

## 5) 还原后状态不符合预期

处理：

- 优先在测试世界复现
- 使用“还原前备份”保留回退点
- 必要时改用指定备份文件恢复

## 6) 如何快速定位“是配置问题还是程序问题”

最短方法：

1. 新建最小配置（1 世界、Full 模式、无过滤）
2. 跑备份和还原闭环
3. 若最小配置成功，问题多半在原配置规则
4. 若最小配置失败，再看环境或程序链路

这个方法可以显著缩短排查时间。
