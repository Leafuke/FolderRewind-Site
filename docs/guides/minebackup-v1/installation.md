# 安装与运行前准备

这页的目标不是“只把程序打开”，而是确保你第一次备份就成功，避免踩路径和权限坑。

## 核心依赖

MineBackup 的核心压缩流程依赖 7-Zip，可通过以下方式提供 `7z.exe`：

- Windows 环境下，MineBackup主程序内置 7-Zip，无需额外安装
- 放在程序同目录（可被自动检测）
- 在设置中手动指定 7-Zip 主程序路径

建议：即便自动检测成功，也在设置页确认一次压缩程序路径，避免后续移动目录后失效。

## 平台说明

- Windows：完整功能（含服务模式）
- Linux / macOS：核心备份与还原可用，但 Windows 服务能力不适用

如果你主要是跨平台使用，优先确认路径分隔符和目录权限。

## 首次启动向导会做什么

首次向导主要完成四件事：

1. 选择语言、字体、主题
2. 选择存档根目录（支持 Java/Bedrock 自动探测）
3. 选择备份目录
4. 确认压缩程序路径

向导结束后建议立即做一次“空注释手动备份”测试，先验证链路再调高级参数。

## 建议的目录规划

- `saveRoot`：仅放要管理的世界目录父路径
- `backupPath`：单独磁盘目录，避免与存档目录混放
- `snapshotPath`：可选，用于热备份临时快照

### 规划示例

- `saveRoot`：`D:/Minecraft/.minecraft/saves`
- `backupPath`：`E:/MineBackup/archives`
- `snapshotPath`：`E:/MineBackup/snapshot-temp`

原则：备份目录与存档目录尽量分盘，降低同盘故障影响。

## 首次运行前检查清单

在点击第一次备份前，确认以下 6 项：

1. `saveRoot` 真实存在且可访问
2. `backupPath` 目录可写
3. 至少有 1 个世界被识别
4. 压缩程序路径有效
5. 无安全软件拦截压缩进程
6. 游戏运行中场景下已规划热备份策略

## 安装后最小自检

1. 能打开设置并看到当前配置
2. 路径项可保存并重启后保留
3. 能成功执行一次最小备份

## 常见安装阶段问题

### 1) 向导里自动探测不到存档目录

- 先手动选择父目录
- 再用“扫描世界”确认是否识别

### 2) 压缩程序看起来可选但备份时报错

- 重新选择一次压缩程序路径并保存
- 避免路径中权限受限目录

### 3) Linux / macOS 下路径看起来正确但不可写

- 使用终端先验证目录写权限
- 尽量使用用户目录下的备份路径
