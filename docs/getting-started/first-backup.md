# 首次备份

本教程会带你完成第一次可验证的备份流程。v1.6.0 开始，你既可以手动创建配置，也可以直接从模板创建。

## 开始前

- 你已经完成 [安装](./installation)
- 你知道备份要保存到哪个目标目录
- 你愿意先用测试数据跑通一次流程

## 路线一：手动创建配置

1. 在首页点击 **新建配置**
2. 选择配置类型
3. 输入配置名称
4. 进入该配置的管理页面
5. 点击 **添加文件夹**，选择要保护的目录

这是最适合第一次理解软件结构的方式。

## 路线二：从模板创建配置

如果你已经有可复用规则，或者官方模板里正好有适合你的场景，可以使用：

1. 在首页点击 **从模板创建**
2. 选择一个本地模板，或先搜索官方模板并导入
3. 确认模板推荐的配置类型
4. 检查自动识别出的来源文件夹
5. 勾选你真正想纳入备份的目录
6. 创建配置

模板适合这些场景：

- 多台电脑使用同一套备份规则
- 为多个项目快速复制相同策略
- 使用插件或特定目录结构时，减少手动配置步骤

## 执行第一次备份

进入配置管理页后，你可以：

- 点击 **一键备份所有**
- 或先选中一个文件夹，再点击 **备份此文件夹**

完成后，到历史页面确认是否已经生成新的备份记录。

## 建议立刻做一次验证

第一次备份完成后，建议至少做下面其中一项：

### 方案 A：手动测试还原

1. 准备一个测试还原目录
2. 从历史记录中选择刚刚生成的备份
3. 执行一次测试还原
4. 检查文件内容是否符合预期

### 方案 B：运行核心功能自动校验

如果你不想只验证单一配置，可以去设置页运行 **核心功能自动校验**。它会自动覆盖更多关键场景，适合在正式启用自动备份前做一次总体验证。

## 模板相关的下一步

如果这套配置已经比较成熟，你可以进一步：

- 在配置设置中点击 **保存为模板**
- 把模板导出到文件，供其他设备使用
- 使用官方模板搜索与导入能力

详细内容见：

- [模板：创建与使用](../guides/templates)
- [模板：分享与导入](../guides/template-sharing)
